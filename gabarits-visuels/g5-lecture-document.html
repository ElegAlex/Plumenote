Infrastructure Procédure · INF-VPN-001

Configurer le VPN Always-On sur poste Windows 11

MZ Rédigé par Mohamed Zemouche
mis à jour il y a 2 h 312 consultations À jour · vérifié il y a 2 h

Contexte et pré-requis

Cette procédure décrit la configuration du VPN Always-On pour les postes utilisateurs DSI et des services de la CPAM 92, dans le cadre de la migration Windows 11 (MUDT). Elle remplace la note interne VPN traditionnel — procédure historique qui reste valable pour les postes non migrés.

L'objectif est d'obtenir une connexion VPN qui démarre automatiquement à l'ouverture de session utilisateur, sans intervention manuelle, et qui couvre tous les flux DSI quel que soit le réseau source (Wi-Fi domicile, partage de connexion, agence).

Astuce. Si votre poste a déjà été migré via l'outil MUDT, l'intégralité des pré-requis est déjà remplie. Seule la configuration du profil Always-On décrite ci-dessous reste à réaliser.

Avant de commencer

  • Le poste doit être joint à l'annuaire national (CPAM*.CRAMIF.LOCAL).
  • Le certificat machine AD-CS 2024 doit être présent dans le magasin local.
  • L'utilisateur doit disposer du rôle DSI-USER-VPN dans l'annuaire (vérifier auprès de Angélique Thibeaudeau).
  • Windows 11 23H2 minimum (vérifier via winver).

Installation du client

Le client VPN est distribué par l'outil de déploiement MUDT et s'installe automatiquement lors de la migration. Pour les postes n'ayant pas encore été migrés, suivre la procédure manuelle ci-dessous.

PowerShell 
# Installation silencieuse du client VPN CPAM
$installer = "\\srv-deploy\mudt$\vpn\CPAMVPNClient-2.4.1.msi"
Start-Process msiexec.exe "/i $installer /quiet /norestart" -Wait

# Vérification
Get-WmiObject -Class Win32_Product | Where-Object { $_.Name -like "*CPAM VPN*" }

Configuration Always-On

La configuration s'effectue en deux temps : installation du certificat machine (1) puis déploiement du profil VPN via GPO (2).

Certificat machine

Le certificat est automatiquement poussé par la GPO GPO-CERT-VPN-ALWAYS-ON au prochain démarrage. Pour forcer la mise à jour :

Terminal 
gpupdate /force /target:computer
certutil -pulse
:: Redémarrer la session

Profil utilisateur

Les paramètres du tunnel sont consignés dans la table de référence ci-dessous. Ces valeurs sont portées par la GPO et ne doivent pas être modifiées manuellement.

Passerellevpn-ao.cpam92.cramif.localRésolution DNS interne

Attention. En cas de déplacement à l'étranger, le profil Always-On peut interdire l'accès à l'internet local. Se reporter à la fiche Déplacements hors métropole — procédure VPN.

Vérification et tests

Après redémarrage, le tunnel doit s'établir automatiquement avant même la saisie du mot de passe utilisateur. Pour vérifier :

  1. Observer l'icône VPN dans la barre système : état « Connecté » en vert.
  2. Ouvrir un terminal et exécuter ping annu-dsi.cpam92.cramif.local.
  3. Vérifier l'accès au portail interne Portail DSI — accueil.

Validé. Si les trois vérifications passent, la configuration est effective. Remonter tout dysfonctionnement au Support N2 Infrastructure.

Dépannage courant

Voir la fiche dédiée VPN ne se connecte pas — dépannage utilisateur. Les cas les plus fréquents sont traités ci-dessous.

Icône VPN rouge au démarrage

Vérifier que le certificat machine n'a pas expiré : certlm.msc, dossier Personnel > Certificats. Si expiré, forcer gpupdate /force.

Connexion instable en 4G / tethering

Le client VPN échoue parfois sur certains opérateurs (NAT symétrique). Forcer le mode TCP en passant par le profil alternatif Always-On (fallback TCP) dans les options avancées.
